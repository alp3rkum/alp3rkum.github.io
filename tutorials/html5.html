HTML Partials

Partial webpages have been a great way to organize your Web project. Many languages and frameworks, such as PHP and ReactJS, offer the ability to make partial webpages. In fact, these two languages are my favorites in Web because of that ability.

But what to do when you can't use any language or framework that offers built-in support for partial webpages? Like when you have to develop a static website using only HTML, CSS and JavaScript? You know, just like this websiteüòÅ

HTML5 used to have a thing called "import" but it has been deprecated, making it harder to make partials... Unless you also know some JavaScript, which is the workaround we will use in this tutorial. Now without further ado, let's learn how to make partials in pure HTML5 & JavaScript.

HTML Code

First, we need to write a basic HTML code to house our partials.

Main Code
<html>
    <head>

    </head>
    <body>
        <div id="partial-container">
        </div>
    </body>
</html>

This code is a basic HTML code that will house our partial, inside a div element that has the "partial-container" ID. This is not a reserved name, and you can name this div anything that you want. Save this code as "index.html".

Now let's create a new JavaScript file. I'll name it "partial.js" for the tutorial. Insert this code in it.

// partial.js
const partialHTML = `<div class="partial"><h1>Hello World!</h1></div>`;
window.onload = function() {
    var partial = document.getElementById("partial-container");
    partial.innerHTML = partialHTML;
}

This code will hold a stringified HTML code in a "partialHTML" constant. Upon the launch of the webpage (window.onload), a function will be triggered that will do the following:

1. Get the div element with the ID "partial-container".

2. Set the innerHTML of the div to the "partialHTML" constant.

Save this code.

Now go back to your index html, and add this script inside the head tags:

<script src="partial.js" ></script>
Save the "index.html" and open it. You will see this (open in new tab to see in full resolution):

Now while this technique might be harder to code than it needs to be, it is a great way to organize your Web project using only HTML, CSS and JavaScript. You can make rather basic header and footer partials this way, which still supports full on CSS styling and DOM manipulation, because the string is written into the page's respective element(s) as HTML code.

With few modifications, you can pretty much establish a primitive routing mechanism using this technique. Can be useful for basic static websites.

Script Defer

JavaScript is a great way to make your website more interactive and have multiple behaviors to perform. Sometimes, it even helps your styling elements to work properly (eg. Bootstrap Dropdown).

In HTML5, we can include JavaScript files in our webpage with this one line of code inside head tags:

<script src="script.js"></script>

This line will make the script file to be loaded as soon as the page is loaded. The page will start executing the commands that script.js file contains, like showing an alert, changing the background color, etc.

But sometimes, this way of loading a script file into the webpage is not feasible, and lack of elements on the webpage will affect how the page looks and performs, as script fails to fully execute. A well-known work around is to write this code at the bottom of body.../body block. This will allow the page to load first, then to execute necessary scripts, and as all elements that are supposed to be in the page are there, the execution of the script file will not yield any problem.

However, there's a third way that combines both two ways, and it's done with adding only one keyword to the code line when we write it inside the head block:

<script defer src="script.js"></script>

This line will tell the browser to execute the script file only when the page is fully loaded. This will allow the complete execution that bottom-of-the-body approach gives, and the ease of writing the inside head approach grants. This will only work in external script loading like this, as internal/inline script is already fully written inside the page.
